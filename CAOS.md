# C/C# 메모리 구조와 저장 방식 정리
---

## `int a = 0;`의 의미

`int a = 0;`는 정수형 변수 `a`를 선언하고 0으로 초기화하는 코드다.

하지만 이 코드에서 더 중요한 건 **값이 0이라는 사실보다, 이 변수가 어디서 선언되었는가**이다.  
같은 `int a = 0;`라도 선언 위치에 따라 저장되는 메모리 영역과 살아있는 시간이 달라진다.

예를 들어 함수 안에서 선언하면 보통 지역 변수로 취급되고, 함수 밖에서 선언하거나 `static`을 붙이면 정적 변수로 취급된다.

### 예시: 함수 안에서 선언한 경우

```c
void Test()
{
    int a = 0;
}
```

이 경우 `a`는 함수가 실행될 때 생성되고, 함수가 끝나면 사라진다.

### 예시: `static`으로 선언한 경우

```c
static int a = 0;
```

이 경우 `a`는 한 번만 생성되고 프로그램이 종료될 때까지 유지된다.

---

## 저장 방식: Automatic, Static, Dynamic

변수는 저장 방식에 따라 크게 세 가지로 나눌 수 있다.

| 종류 | 의미 | 주로 저장되는 위치 | 특징 |
|---|---|---|---|
| Automatic | 함수 안에서 자동으로 생성되는 변수 | Stack | 함수가 끝나면 사라진다 |
| Static | 한 번 생성되고 계속 유지되는 변수 | Data / BSS | 프로그램 종료 전까지 유지된다 |
| Dynamic | 실행 중 필요할 때 생성하는 데이터 | Heap | 직접 해제하거나 런타임이 정리한다 |

### Automatic

자동 변수는 함수 안에서 선언한 일반적인 지역 변수다.  
함수가 호출될 때 스택에 생성되고, 함수가 끝나면 자동으로 제거된다.

```c
void Attack()
{
    int damage = 15;
}
```

위 코드의 `damage`는 `Attack()` 함수가 실행되는 동안만 존재한다.

### Static

정적 변수는 한 번만 생성되고 프로그램이 끝날 때까지 유지된다.  
함수 안에 선언해도 호출이 끝난 뒤 사라지지 않고 값을 계속 기억한다.

```c
void Counter()
{
    static int count = 0;
    count++;
}
```

이 함수는 호출할 때마다 `count` 값이 누적된다.

### Dynamic

동적 메모리는 실행 중 필요에 따라 생성하는 메모리다.  
C에서는 `malloc()`으로 할당하고 `free()`로 해제한다.

```c
int* arr = (int*)malloc(sizeof(int) * 10);
free(arr);
```

이 코드는 정수 10개를 저장할 수 있는 메모리를 힙에 만들고, 사용이 끝난 뒤 해제한다.

C#에서는 보통 `new`를 사용해 객체를 만들고, 해제는 가비지 컬렉터가 담당한다.

```csharp
int[] numbers = new int;[3]
```

## 프로그램 메모리 구조

프로그램이 실행될 때 메모리는 여러 영역으로 나뉘어 사용된다.

대표적으로 다음과 같은 영역이 있다.

- Code(Text) 영역
- Data 영역
- BSS 영역
- Heap 영역
- Stack 영역

---

## Code(Text) 영역

Code 영역은 실제 실행할 명령어가 들어가는 공간이다.  
함수의 본문이나 프로그램 로직이 기계어 형태로 저장된다.

쉽게 말하면, 프로그램이 “무엇을 할지”에 대한 실행 코드가 들어 있는 장소다.

---

## Data 영역

Data 영역은 **초기값이 있는 전역 변수나 static 변수**가 저장되는 곳이다.

예를 들어 아래 변수는 Data 영역에 들어간다.

```c
static int a = 10;
```

초기값이 이미 정해져 있기 때문에 프로그램 시작 시부터 해당 값이 함께 준비된다.

---

## BSS 영역

BSS 영역은 **초기값이 없는 전역 변수나 static 변수**가 저장되는 곳이다.

예를 들어 아래 변수는 BSS 영역에 들어간다.

```c
static int a;
```

초기값을 따로 쓰지 않아도 프로그램 시작 시 자동으로 0으로 초기화된다.

즉, Data와 BSS의 차이는 **초기값이 명시되어 있느냐**라고 보면 된다.

---

## Stack 영역

Stack은 함수 호출과 함께 사용되는 임시 메모리 공간이다.

여기에는 주로 다음이 들어간다.

- 지역 변수
- 매개변수
- 함수가 끝난 뒤 돌아갈 위치 정보

함수가 실행되면 쌓이고, 함수가 끝나면 가장 최근에 들어간 것부터 제거된다.

```c
void Move()
{
    int speed = 5;
}
```

이 코드에서 `speed`는 Stack에 저장된다.

Stack은 빠르게 만들고 없앨 수 있어서 함수 안의 임시 데이터 관리에 적합하다.

---

## Heap 영역

Heap은 실행 중에 필요할 때 직접 확보하는 메모리 공간이다.

크기가 유동적인 데이터나 오래 유지해야 하는 데이터는 보통 Heap을 사용한다.

```c
int* hp = (int*)malloc(sizeof(int));
*hp = 100;
free(hp);
```

또는 C#에서는 객체 생성 시 Heap을 많이 사용한다.

```csharp
Player player = new Player();
```

Heap은 유연하지만 관리 비용이 더 든다.  
C에서는 직접 해제하지 않으면 메모리 누수가 생길 수 있고, C#에서는 가비지 컬렉터가 정리하는 대신 성능에 영향을 줄 수 있다.

---

## Stack과 Heap의 차이

Stack과 Heap은 둘 다 메모리지만 목적이 다르다.

| 항목 | Stack | Heap |
|---|---|---|
| 생성 방식 | 함수 호출 시 자동 생성 | 실행 중 직접 요청 |
| 제거 방식 | 함수 종료 시 자동 제거 | 직접 해제 또는 GC 정리 |
| 용도 | 지역 변수, 매개변수 | 객체, 배열, 동적 데이터 |
| 특징 | 빠르고 단순함 | 유연하지만 관리 비용이 큼 |

Stack은 잠깐 쓰는 임시 작업 공간이고, Heap은 필요할 때 크게 꺼내 쓰는 창고 같은 개념으로 이해하면 편하다.

## `static` 변수의 동작 방식

`static`은 “매번 새로 만드는 변수가 아니라, 하나를 계속 유지해서 쓰는 변수”라고 이해하면 쉽다.

일반 지역 변수는 함수가 호출될 때마다 새로 생성되지만, static 변수는 한 번만 만들어지고 값이 유지된다.

```c
void PrintCount()
{
    static int count = 0;
    count++;
    printf("%d\n", count);
}
```

이 함수를 여러 번 호출하면 `count` 값이 계속 증가한다.  
만약 `static`이 없었다면 함수가 호출될 때마다 `count`는 다시 0부터 시작했을 것이다.

## C와 C#의 실행 방식 차이

칠판에서는 C와 C#이 실행되는 방식의 차이도 함께 설명하고 있다.

두 언어 모두 결국 기계어로 실행되지만, 그 과정이 다르다.

---

## C의 실행 과정

C는 소스코드를 컴파일한 뒤, 바로 기계어에 가까운 네이티브 코드로 변환하여 실행한다.

흐름을 단순하게 적으면 다음과 같다.

```text
C 코드 -> 컴파일 -> 기계어 -> 실행
```

즉, 실행 전에 미리 CPU가 이해할 수 있는 코드로 만들어 두는 방식이다.

---

## C#의 실행 과정

C#은 보통 바로 기계어가 되지 않고, 먼저 중간 언어인 IL(Intermediate Language)로 변환된다.

그 후 .NET 런타임이 이 IL을 실행 시점에 기계어로 바꾸어 실행한다.

```text
C# 코드 -> IL(중간 언어) -> JIT 컴파일 -> 기계어 -> 실행
```

이 구조 덕분에 C#은 다양한 플랫폼에서 비교적 유연하게 동작할 수 있다.

---

## IL과 JIT

### IL

IL은 C# 코드를 바로 CPU가 읽을 수 있는 기계어로 만들기 전에 거치는 중간 단계의 코드다.

완전히 사람이 읽기 쉬운 코드는 아니지만, 아직 특정 CPU 전용 기계어도 아니다.

### JIT

JIT는 Just-In-Time Compiler의 줄임말이다.

이건 IL 코드를 프로그램 실행 도중 필요한 시점에 기계어로 바꾸는 역할을 한다.

즉, C#은 다음 순서로 실행된다고 보면 된다.

1. 사람이 C# 코드를 작성한다.
2. 컴파일하면 IL이 만들어진다.
3. 실행할 때 JIT가 IL을 기계어로 바꾼다.
4. CPU가 그 기계어를 실행한다.

---

## C와 C#의 차이를 예시로 보기

### C

```c
#include <stdio.h>

int main()
{
    int a = 10;
    printf("%d\n", a);
    return 0;
}
```

이 코드는 컴파일 시 네이티브 코드로 변환되어 실행된다.

### C#

```csharp
using System;

class Program
{
    static void Main()
    {
        int a = 10;
        Console.WriteLine(a);
    }
}
```

이 코드는 먼저 IL로 컴파일되고, 실행할 때 JIT가 기계어로 바꿔 실행한다.

---

## C#에서 이 개념을 이해해야 하는 이유

유니티에서 C#을 사용할 때도 이 개념은 중요하다.

예를 들어:

- 지역 변수는 함수 안에서 잠깐 쓰는 값이다.
- `static` 변수는 여러 객체가 함께 공유할 수 있다.
- `new`로 만든 객체는 보통 Heap에 생성된다.
- 값이 왜 유지되는지, 왜 공유되는지, 왜 성능 문제가 생기는지를 이해할 때 메모리 구조 지식이 필요하다.

### 예시

```csharp
using UnityEngine;

public class Test : MonoBehaviour
{
    static int sharedCount = 0;
    int localCount = 0;

    void Start()
    {
        sharedCount++;
        localCount++;

        Debug.Log($"shared: {sharedCount}, local: {localCount}");
    }
}
```

이 코드에서 `sharedCount`는 static이므로 여러 객체가 함께 공유할 수 있다.  
반면 `localCount`는 각 객체가 따로 가진다.

이 차이를 이해하지 못하면 “왜 값이 같이 올라가지?” 같은 버그를 만났을 때 원인을 찾기 어려워진다.
